# 設計文件：手動調整場地數量 + 場上選手替換（混合方案）

日期：2026-06-12
來源需求：孟娜 2026-06-11 對話（6/25 實測多一面場地；太常出現與前一場重複 3–4 人的組合需手動處理）

## 背景

- 場地數量目前寫死於 `js/script.js:5`：`courts = [[], [], []]`。其餘邏輯均以 `courts.length` 迭代，動態增減場地不需改排場演算法。
- 手動模式（`isManualMode`）下場地卡片已有「清空」（`manualEndMatch`）與每位選手的「休息」（`restPlayerOnCourt`）按鈕。
- 「休息」按鈕語意：下場去休息區＋記一場比賽＋按 ABC 等級補人。本次新增的「替換」語意不同：**這場沒上**，回預備區、等待輪次 +1、完全隨機補人。

## 功能一：手動調整場地數量

### UI
- 場地區塊標題旁新增控制列：`場地數量 − 3 ＋`（仿現有等待輪次 +/- 按鈕樣式）。
- 範圍 1–6 面，超出範圍按鈕無效。
- 不限手動模式，自動模式也可調整。

### 行為
- **增加**：`courts.push([])`，立即重繪場地區。
- **減少**：固定移除**最後一面**場地。若該場地上有選手，擋下並 `alert` 提示「場地 N 還有選手，請先清空該場地」。
- 調整後呼叫 `updateCourtsDisplay()`、`updateNextMatchPrediction()`（場地數影響下一場預告）、`saveGameState()`。

### 存檔相容
- `courts` 陣列本身已在 `saveGameState()` 中保存，長度隨之保存，無需新欄位。
- `loadGameState()` 還原時以存檔中的 `courts` 長度為準（現有行為即可，驗證即可）。

## 功能二：場上選手替換（混合方案一＋方案二）

僅在**手動模式**顯示，兩個入口共用同一核心函式。

### 核心函式 `swapPlayerOnCourt(courtIndex, playerName, excludeNames)`

1. 從 `courts[courtIndex]` 找到該選手，記住其**索引位置**（保持隊伍分組：前 2 人為隊伍一、後 2 人為隊伍二）。
2. 從預備區隨機抽一位補位人選：
   - 抽樣池 = `readyPlayers` 全體（**完全隨機，含「剛下場」者**），但排除 `excludeNames`（本次替換中已被換下的人，避免換下後立刻被抽回）。
   - 池為空 → 不執行，回傳失敗。
3. 被換下的選手：
   - `waitingTurns = (waitingTurns || 0) + 1`（這場沒上，等待累加）。
   - 清除 `justFinished`、`justJoinedReady` 標記（預備區直接顯示「等待 X 場」，比照 `adjustWaitingTurns` 的處理）。
   - **不**增加 `matches`（沒有實際上場）。
   - push 回 `readyPlayers`。
4. 補上的選手：
   - 從 `readyPlayers` 移除，`waitingTurns = 0`、清除 `justFinished`/`justJoinedReady`，放入場地陣列**原索引位置**（`splice` 置入）。
   - **不**動 `matches`（比賽結束時由既有 `endMatch`/`manualEndMatch` 流程處理）。
5. 場地 `startTime` 不變。
6. 結尾統一：`updateLists()`、`updateCourtsDisplay()`、`updateNextMatchPrediction()`、`saveGameState()`。

### 入口一（方案二）：單人「替換」按鈕
- 位置：手動模式下，場地卡片中每位選手「休息」按鈕旁新增「替換」。
- 行為：`swapPlayerOnCourt(courtIndex, name, [name])` 一次。
- 防呆：預備區（排除被換者）無人可補 → `alert("預備區沒有人可以補位")`，不動作。

### 入口二（方案一）：整組「替換組合」按鈕
- 位置：手動模式下，場地標題列「清空」按鈕旁新增「替換組合」。
- 行為：
  1. 場地必須滿 4 人，否則提示。
  2. 預備區人數 < 2 → 提示不足，不動作。
  3. 從場上 4 人隨機抽 2 人，依序呼叫核心函式；`excludeNames` 累積兩位被換下者，確保補位不會抽到剛被換下的任何一人，且兩個補位人選不重複。
- 不需 confirm（操作可再按一次重抽，成本低）。

### 隨機性
- 使用 `Math.random()` 均勻抽樣（瀏覽器端執行，無測試框架限制）。

## 不做的事（YAGNI）
- 替換時不做技能等級 ±1.5 配對檢查（孟娜明確要求「完全隨機、不用思考」）。
- 不做替換的語音播報（手動模式現場操作，必要時之後再加）。
- 不做替換歷史記錄。

## 測試方式

依專案慣例以瀏覽器手動測試：
1. 場地數 3→4→6（上限擋住）→ 減到有選手的場地（擋住提示）→ 清空後可減 → 重新整理後數量保留。
2. 手動排場 4 人上場 → 按「替換組合」：恰 2 人回預備區顯示「等待 +1」、2 位隨機新人補上、隊伍位置正確。
3. 單人「替換」：該人回預備區等待 +1，隨機 1 人補上，且不會抽回他自己。
4. 預備區 0 人／1 人時兩種按鈕的防呆提示。
5. 換下「剛下場」標記者，確認顯示變為「等待 X 場」而非「剛下場」。
